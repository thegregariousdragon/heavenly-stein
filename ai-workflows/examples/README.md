# Story Pipeline Examples

These examples show how to use the AI workflow files during story development.

## Canon status

These are non-canon workflow examples.

Do not treat these examples as active story events, active chronology, active scene blocking, or promoted canon.

They exist only to demonstrate process.

## Files

- `station-scene-intake-example.md`: shows a station-based story intake.
- `off-station-scene-intake-example.md`: shows an off-station story intake.
- `character-voice-check-example.md`: shows how to report character voice and competence checks.
- `continuity-audit-example.md`: shows how to report a continuity audit.

## How to use these examples

Use them as format guides, not fact sources.

A future AI assistant may copy the structure of these examples, but should not copy the fictional details into a real story unless the user explicitly asks for them and canon promotion is handled separately.
