# Stories

This folder is for story prose packages.

## Status rules

- Story prose in this folder may be draft, publishable, archived, or reference-only depending on its local README and status notes.
- Story prose does not automatically promote every detail inside it to canon.
- Durable canon facts still belong in `canon/` and should be promoted through the canon process.
- When a story is treated as publishable, its package should include a `canon-status.md` file that separates locked canon facts, candidate-only details, and prose-only choices.

## Current subfolders

- `publishable/` — story packages intended for publication or publication review.

## Reader artifacts

Use `ai-workflows/EPUB_PUBLISHING.md` when generating reader-facing EPUB files from publishable story Markdown.

Generated EPUBs are local artifacts by default. Do not commit generated EPUB files back to the repo unless the author explicitly asks.
