## Ambiguity rules

- If C could mean Clara/Klara or another C-name in a specific context, ask.
- If J could mean Janelle or Jun Park from Across the Table, default to Janelle unless the request is clearly about food/hospitality media.
- If M could mean Matthias or Mara, default to Matthias only in family context; otherwise ask.
- If Mara is used without a surname, ask whether the user means Mara Venn or Captain Mara Sol unless the context clearly resolves it.
- If RouteCast is used during an urgent hazard, BeaconWatch may be the format. If the distinction matters, ask.
- If the request says "use my shortcuts," apply this file before asking clarifying questions.

## Name and pronoun usage policy

This policy supersedes looser nickname guidance. In story prose, use a character's full name only when that character is introduced in that story, or when a formal record, official report, public statement, roster, legal document, or deliberate formal address requires it. After introduction, use the first name or common name.

If a character has a quoted name in their formal listing, such as Heinrich “Heinz” Schmitt, Felicia “Feli” Brandt, or Nkiru “Nikki” Adebayo, the quoted name is normally the everyday name used in narration and dialogue. The unquoted first name is more formal and should be reserved for official records, family/formal address, or deliberate emotional emphasis.

Check character pronouns against canon or quick-reference cards before drafting. Do not infer pronouns from names, roles, appearance, or older flawed drafts. Treat misgendering as a continuity error.
