# Story Request Template

This template defines the default story-generation workflow for The Heavenly Stein / Heinz Schmitt stories.

Use it as a working aid, not a burden. Greg may provide a full template, a shortened version, or a casual prompt with shortcuts. When the prompt is clear enough, proceed. Ask only high-impact missing questions before drafting.

For shortcuts, use `quick-reference/SHORTHAND_DECODER.md`.

## Default assistant behavior for story generation

When generating a story, outline, scene, revision, or continuity audit:

1. Decode Greg's shortcuts using `SHORTHAND_DECODER.md`.
2. Identify the main story type, main cast, location, tone, and canon systems being tested.
3. Ask only missing questions that would materially change the story.
4. Do not ask for information already supplied in the prompt or in the canon package.
5. If the brief is clear enough, proceed with the requested output.
6. Use the dense numbered canon as source of truth; use quick-reference files as handles.
7. Treat character fingerprints as guidance, not mandatory tics.
8. Keep Heinz important without making him the default solution to every ensemble problem.
9. Respect mechanics: transport, clean-side/restricted-side routing, force escalation, weapons limits, communication limits, law/jurisdiction, and rebuild-era safety review culture.
10. When a proposed story would create new hard canon, flag that before or after the draft.

## Typical high-impact questions

Ask these only when missing and important.

### 1. What kind of story is this?

Examples: security incident, food/hospitality story, governance dispute, family scene, ship action, media-pressure episode, artifact problem, Heinz/Feli relationship beat.

### 2. Who must carry the solution?

Examples: Janelle leads and Heinz supports; Tariq notices the problem first; Feli controls public messaging; Rolf solves the mechanical cause; Dr. Santos identifies the real danger; Amara controls movement; Greta forces review afterward.

### 3. What canon system should be tested?

Examples: weapons limits, data slate communication, clean-side/restricted-side routing, law enforcement review, Guild rights, Flight Control authority.

### 4. How hard should the consequences land?

Options: mostly comic, inconvenient but safe, serious but no deaths, emotionally heavy, public scandal, legal aftermath, physical injuries, long-term canon change.

### 5. Should this create new canon?

Options: exploratory only, canon if it works, intended canon, not canon.

## Story-generation guardrails

- Do not use quick-reference files to override numbered canon.
- Do not flatten character fingerprints into catchphrases.
- Do not make competent people stupid for comedy.
- Do not let media appear physically on station without a reason.
- Do not allow direct teleportation into public spaces unless canon deliberately creates an exception and consequences.
- Do not use Heinz's Scatterbeam casually in public spaces.
- Do not make nonlethal weapons harmless.
- Do not make ship disabling easy unless conditions justify it.
- Do not turn grief, deaths, family harm, or private trauma into content just because it helps the station.
- Do not treat Der Himmelskrug as a sovereign state; it is privately founded, partner-backed, and IFP-recognized civic/route-safety infrastructure.

## Name/pronoun rule

Use full character names only at story introduction or in formal records/address. After introduction, use first names or common quoted names. Check canon for pronouns before drafting; treat misgendering as a continuity error.
